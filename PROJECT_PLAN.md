# 校园轻集市 - 项目规划文档

## 一、页面清单

基于需求分析，本项目需开发以下页面：

### 核心页面（9个）

| 序号 | 页面名称 | 路由路径 | 说明 |
|------|----------|----------|------|
| 1 | 身份创建页 | `/setup` | 用户首次进入系统，填写本地身份信息 |
| 2 | 今日集市首页 | `/` | 展示系统概览、功能入口、统计数据 |
| 3 | 集市信息列表页 | `/market` | 浏览所有校园信息，支持筛选排序 |
| 4 | 信息详情页 | `/detail/:id` | 展示单条信息完整内容 |
| 5 | 信息发布页 | `/publish` | 发布二手/失物招领/拼单/跑腿信息 |
| 6 | 消息中心页 | `/messages` | 展示会话列表和聊天记录 |
| 7 | 个人中心页 | `/profile` | 用户档案、我的发布、我的收藏 |
| 8 | 趋势看板页 | `/trends` | 数据可视化展示 |
| 9 | 安全提醒组件 | - | 展示交易安全提示（非独立页面） |

## 四、开发重点

### 重点一：数据层设计 ⭐⭐⭐

**为什么是重点**：
- 所有功能都依赖数据交互
- 数据模型设计影响整个项目架构
- JSON Server 的 CRUD 操作是核心

**实现要点**：
```typescript
// 数据模型设计
interface MarketInfo {
  id: string
  type: 'secondhand' | 'lostfound' | 'groupbuy' | 'errand'
  title: string
  campus: string
  location: string
  description: string
  tags: string[]
  images: string[]
  status: 'active' | 'completed' | 'closed'
  publisher: {
    id: string
    nickname: string
    avatar?: string
  }
  createdAt: string
  viewCount: number
  favoriteCount: number
  
  // 类型专属字段
  price?: number          // 二手
  quality?: string        // 二手成色
  lostFoundType?: 'lost' | 'found'  // 失物招领
  targetCount?: number    // 拼单目标人数
  currentCount?: number   // 拼单当前人数
  deadline?: string       // 截止时间
  reward?: number         // 跑腿酬劳
}

interface Conversation {
  id: string
  participants: string[]
  lastMessage: Message
  unreadCount: number
  relatedInfoId: string
  updatedAt: string
}

interface Message {
  id: string
  conversationId: string
  senderId: string
  content: string
  type: 'text' | 'bargain'  // 普通消息或砍价消息
  createdAt: string
}
```

### 重点二：状态管理架构 ⭐⭐⭐

**为什么是重点**：
- 收藏状态需要在多页面同步
- 消息未读数需要实时更新
- 用户信息需要在全局使用

**Pinia Store 设计**：
```typescript
// stores/userStore - 用户状态
{
  user: User | null
  isLoggedIn: boolean
}

// stores/marketStore - 市场信息
{
  infoList: MarketInfo[]
  currentFilter: FilterOptions
  currentSort: SortOptions
  isLoading: boolean
}

// stores/favoriteStore - 收藏状态
{
  favoriteIds: string[]  // 本地收藏的ID列表
}

// stores/messageStore - 消息状态
{
  conversations: Conversation[]
  currentConversation: Conversation | null
  messages: Message[]
  totalUnread: number
}
```

### 重点三：路由与权限控制 ⭐⭐⭐

**为什么是重点**：
- 未创建身份时需要引导到设置页
- 用户操作需要携带身份信息
- 消息和收藏需要关联用户

**路由守卫设计**：
```typescript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 需要身份验证的页面
  if (to.meta.requiresAuth && !userStore.isLoggedIn) {
    next('/setup')
  } else {
    next()
  }
})
```

### 重点四：表单与校验 ⭐⭐⭐

**为什么是重点**：
- 发布表单是核心功能之一
- 不同类型表单字段不同
- 需要统一的校验机制

**表单设计策略**：
```
发布页设计模式：
1. 顶层类型选择器（单选卡片）
2. 根据类型动态渲染表单组件
3. 统一使用基础校验规则
4. 提交前进行完整校验
```

### 重点五：Mock 数据与模拟逻辑 ⭐⭐

**为什么是重点**：
- 砍价模拟需要简单规则
- 消息回复需要自动生成
- 数据需要有真实性

**模拟规则设计**：
```typescript
// 砍价模拟回复规则
function generateBargainReply(offer: number, price: number) {
  const ratio = offer / price
  if (ratio >= 0.8) return '可以便宜一点，你看...'
  if (ratio >= 0.6) return '这个价格有点低了...'
  if (ratio >= 0.4) return '真的不能这么低，最多...'
  return '这个价格我不卖'
}

// 消息模拟回复规则
function generateAutoReply(message: string) {
  const replies = [
    '好的，我考虑一下',
    '可以，具体什么时候方便？',
    '好的，在哪里交易？',
    '嗯嗯，我知道了'
  ]
  return replies[Math.floor(Math.random() * replies.length)]
}
```

### 重点六：ECharts 图表集成 ⭐⭐

**为什么是重点**：
- 趋势看板是展示型功能
- 需要直观的数据可视化
- 图表类型选择要合适

**图表配置要点**：

集市信息统计：

- 校区分布：柱状图

## 六、风险与建议

### 潜在风险

1. **JSON Server 数据持久化**
   - 问题：刷新页面数据重置
   - 建议：使用 json-server 的 `--watch` 参数，或部署到云端

2. **表单动态渲染复杂度**
   - 问题：四种类型表单字段不同
   - 建议：使用 Vue 的动态组件 `component :is`

3. **状态同步问题**
   - 问题：收藏状态在多页面可能不同步
   - 建议：统一使用 Pinia 管理，通过 localStorage 持久化

### 开发建议

1. **先数据后界面**：先设计好数据模型和 Mock 数据，再开发界面
2. **组件复用**：抽取通用组件（卡片、筛选栏、弹窗）
3. **渐进式开发**：先完成核心流程，再补充细节功能
4. **及时测试**：每个功能完成后进行测试验证
